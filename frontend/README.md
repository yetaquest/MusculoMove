# MusculoMove Frontend

This frontend is a Vite + React + TypeScript viewer for the MusculoMove backend. It no longer depends on `public/models/avatar.glb` as the primary render path. Instead it loads the OpenSim-derived GLTF generated by the backend at `/api/viewer/model.gltf` and applies backend segment transforms directly onto converted OpenSim body nodes.

## Run it

1. Start the backend from the repo root:

```bash
uv run --cache-dir /tmp/uv-cache python main.py serve --host 127.0.0.1 --port 8000
```

2. In `frontend/`, install dependencies if needed:

```bash
npm install
```

3. Start the frontend dev server:

```bash
npm run dev
```

The Vite dev server proxies `/api/*` requests to `http://127.0.0.1:8000`.

## Viewer behavior

1. The frontend fetches `/api/manifest`.
2. It reads `viewer.asset_url` and `viewer.body_nodes`.
3. It loads the generated GLTF from `/api/viewer/model.gltf`.
4. It applies backend segment transforms to matching OpenSim body nodes such as `Body:/bodyset/pelvis`.
5. If the GLTF cannot be loaded, the app falls back to debug geometry and keeps the last valid pose visible.
